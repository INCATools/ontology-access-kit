Ontobee Endpoint

Specializes the sparql implementation to allow access for ontologies on the Ontobee linked data server.

ntobee is the default linked data server for most OBO Foundry library ontologies. Ontobee has also been used for many non-OBO ontologies.

Initialization

Access to all ontologies on Ontobee is via a selector with syntax ontobee: (i.e. specify the ontobee scheme with no slug)

To access a specific ontology on ontobee follow the scheme with the ontology ID (all in lowercase). E.g.

runoak -i ontobee:mp COMMAND [COMMAND-OPTIONS]

Wraps the Ontobee sparql endpoint

An OntobeeImplementation can be initialed by:

>>>  oi = OntobeeImplementation()

The default ontobee endpoint will be assumed

Alternatively, use a selector:

>>> oi = get_implementation_from_shorthand("ontobee:")
>>> for a in oi.ancestors("UBERON:0002398", predicates=[IS_A]):
>>>     ...

See: https://www.ontobee.org/
